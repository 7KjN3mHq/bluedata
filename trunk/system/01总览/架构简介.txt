优化大师项目测试架构

                                (192.168.1.240)ld1(192.168.0.240)      app1(192.168.0.241)
client  ->  VIP(192.168.1.246) |            heartbeat            |----|                   |---- db(192.168.0.245)
                                (192.168.1.250)ld2(192.168.0.250)      app2(192.168.0.242)

说明：
调度器 ld1 ld2 运行 lvs 调度真实的 web 服务器。
调度器只能有一个是 active 的，因为是一个虚拟 IP(192.168.1.246) ，同时 active 会造成冲突。
ld1 down 掉后，ld2 检测到 ld1 的心跳停止后会启用虚拟 IP 并接管调度任务。
真实 web 服务器其实也是 ld1 ld2 ，运行了 nginx 直接响应静态请求。
动态的 php 请求转给了 app1 和 app2 这两部应用服务器。
真实 web 服务器和应用服务器都是同时 active 的，一台 down 掉不会影响正常访问。
数据库是 db 服务器，接受来自 app1 和 app2 的数据库查询请求。
存储也放在了 db 上，通过 nfs 把 /home 目录共享给了 ld1 ld2 app1 app2
除了对外提供服务的真实 web 服务器有公网 IP(192.168.1.0/24) ，集群内使用私有网段 192.168.0.0/24
web 服务的根目录在 /home/www ，php 的 session 默认存放在了 /tmp ，
这会造成 session 的不同步，改为了放到 /home/phpsession 下。
