解决办法：
dd if=/dev/zero bs=1M count=1 of=/dev/hda7; sync
drbdadm create-md data

配置 LVM
vi /etc/lvm/lvm.conf
# filter = [ "a/.*/" ]
filter = [ "r|/dev/cdrom|", "r|/dev/hda7|", "a|/dev/drbd0|" ]

把一个节点设置为主节点
drbdsetup /dev/drbd0 primary -o

然后创建 pv/vg/lv
pvcreate /dev/drbd0
vgcreate vg1 /dev/drbd0
lvcreate -L 4G -n lv1 vg1
lvcreate -l 1595 -n lv2 vg1

mkfs -t ext3 /dev/vg1/lv1
mkfs -t ext3 /dev/vg1/lv2

mount -t ext3 /dev/vg1/lv1 /data
mount -t ext3 /dev/vg1/lv2 /home

cd /data
echo "test" > test

umount /data
umount /home
vgchange -a n vg1
drbdadm secondary r0


drbdadm primary r0
vgscan    # 第一次需要，以后就不用再 vgscan 了。
vgchange -a y vg1
mount -t ext3 /dev/vg1/lv1 /data
mount -t ext3 /dev/vg1/lv2 /home

cat /data/test
test

cd /home
echo "test" > test

交换上面的步骤，在 node1 的 /home 目录下应该也能看到 test 文件。

调整容量：

umount /home

       degr-wfc-timeout time
              Wait for  connection  timeout,  if  this  node  was  a
              degraded cluster.  In case a degraded cluster (= clus-
              ter with only one node left) is rebooted, this timeout
              value is used instead of wfc-timeout, because the peer
              is less likely to show up in time, if it had been dead
              before.  Default is 60, unit is seconds. Value 0 means
