1. 原理
以关键词和 ID 号散列出一个用于搜索的表。
当用户输入查询字段时，对内容进行拆词，
去掉单字后把每个词与搜索表中的关键词
进行匹配，然后得到 ID 号及数量。

2. 搜索数据的存储
建立存放关键词和 ID 的对应表。
以建立 goods 表的搜索为例：
先建立一个 goods 关键词表，
关键词以分词程序使用的词典为基准，再添加建材行业特有的关键字。
再从 goods 表中读取数据，分词后存入 search 的对应表。
假定有这样一条记录：
id     name                        生产商                产地  经销商                    keywords
12345  德尔奥美系列钻石水晶面地板  苏州德尔地板有限公司  苏州  德尔地板成都富森销售中心  德尔，地板，奥美，钻石，水晶面

其它字段不作为搜索字段，就不需要了。

取出这条记录时，先把 keywords 和产地的内容取出，
再取出 name 生产商 经销商 字段的内容，用分词程序拆分，
这个时候也许会拆分出以下词语：
德尔 奥美 系列 钻石 (水晶|水晶面) 地板
苏州 (德尔 地板|德尔地板) (有限 公司|有限公司)
(德尔 地板|德尔地板) 成都 富森 (销售 中心|销售中心)

将重复的词去掉后，得到这样一个关键词数组：
德尔 奥美 系列 钻石 水晶 水晶面 地板 苏州 德尔地板 有限 公司 有限公司 成都 富森 销售 中心 销售中心

然后把这个关键词数组与 goods 关键词表中的关键词进行匹配，在匹配关键词
的 ids 字段添加这条记录的 id - 12345 并在关键词表的数量字段 +1 。

3. 取出搜索结果
当用户进行搜索时，搜索系统做如下的工作：
分词 -> 单词与搜索表的关键词作 "=" 匹配 -> 取出对应 id -> 组合结果输出

可以方便的列出切分出的各关键词是什么并标记出这个关键词有多少条相符的记录。
